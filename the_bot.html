How to use the Routing Desk
For i-lab staff — live routing in founder conversations
Step 1
Capture what you're hearing
Select chips for the same 5 questions a founder answers in Find Your Path:
  • Stage — Explore / Test / Propel
  • Goal — feedback, milestones, community, or funding
  • Time — light, moderate, or heavy commitment
  • Sector — Life Sci, Climate, Deep Tech, Social Impact, or General
  • Format — Remote, Hybrid, or In-Person
The top "Eligibility check" row is staff-only — it never affects the persona or routing, since founders never answer it.
Step 2
Read the routing banner
The Routing as: line names the exact persona this founder would land on in Find Your Path, plus the sector/format used (defaults shown explicitly when unselected — same defaults the quiz itself falls back to).
Step 3
Use the cards live in conversation
Start right here = the persona's 3 steps. Also relevant = matched events. Advisors to loop in = tagged experts. Office hours = tagged sessions — step-linked ones are promoted automatically. All four sections pull live from ilab-data.js.
For editors
This file has no data of its own
Unlike the old version of this tool, there is no PROGRAMS array here anymore.
  • To change content (events, steps, experts, office hours, persona copy) — edit via data-editor.html, export, and push the new ilab-data.js.
  • To change persona scoring — edit calcPersonaFromAnswers() inside ilab-data.js. It's called by both this tool and the founder quiz — one change updates both, guaranteed.
  • To change chip labels — edit SIGNAL_CONFIG in this file, but never touch a chip's value — those must exactly match the founder quiz's codes.
Single source of truth: ilab-data.js
Why this matters
Guaranteed parity, not coincidental parity
The previous version of this tool had its own hardcoded program list and its own scoring logic — they could (and did) drift from what founders actually see. This version calls the exact same functions the founder-facing quiz calls, so a staff member and a founder giving identical answers will always get identical output.
